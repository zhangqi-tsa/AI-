## ADDED Requirements

### Requirement: Noise config generator reads review report
`gen-noise-config.py` SHALL parse a Keploy review report Markdown file to extract dynamic field names and generate Keploy-compatible noise configuration.

#### Scenario: Valid review report
- **WHEN** `gen-noise-config.py --review-report reports/keploy-review.md --output keploy/config.yaml` is executed with a report containing dynamic fields
- **THEN** the script SHALL generate a YAML file with noise rules for each detected dynamic field

#### Scenario: No dynamic fields in report
- **WHEN** the review report contains "No dynamic fields detected."
- **THEN** the script SHALL output an empty noise config and print "No noise rules needed"

### Requirement: Noise config maps fields to Keploy noise paths
`gen-noise-config.py` SHALL map detected dynamic field names to appropriate Keploy noise config paths based on common patterns.

#### Scenario: Timestamp field mapping
- **WHEN** a `timestamp` dynamic field is detected
- **THEN** the generated config SHALL include `resp.body.timestamp` and `resp.header.Date` as noise paths

#### Scenario: UUID/trace_id field mapping
- **WHEN** a `uuid` or `trace_id` dynamic field is detected
- **THEN** the generated config SHALL include `req.header.X-Request-Id` and `resp.header.X-Request-Id` as noise paths

#### Scenario: Custom field name
- **WHEN** a dynamic field name does not match any known pattern
- **THEN** the generated config SHALL include `resp.body.<field_name>` as a noise path

### Requirement: Noise config output is valid Keploy format
`gen-noise-config.py` SHALL produce output in Keploy's native noise config YAML format.

#### Scenario: Output format
- **WHEN** the noise config is generated
- **THEN** the output SHALL follow Keploy's config.yaml schema with `assertions.noise` as a list of strings

### Requirement: Noise config marks auto-generated content
`gen-noise-config.py` SHALL add a comment header to the generated config indicating it was auto-generated and requires human review.

#### Scenario: Comment header
- **WHEN** the config file is generated
- **THEN** the file SHALL begin with `# AUTO-GENERATED by gen-noise-config.py — REVIEW REQUIRED`

### Requirement: Noise config does not overwrite existing config
`gen-noise-config.py` SHALL refuse to overwrite an existing config file unless `--force` is specified.

#### Scenario: Existing config without force
- **WHEN** the output file already exists and `--force` is not provided
- **THEN** the script SHALL report the existing file and exit 1

#### Scenario: Existing config with force
- **WHEN** `--force` is provided and the output file exists
- **THEN** the script SHALL overwrite the file
